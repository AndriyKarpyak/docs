  2. class MetadataTable extends slick.jdbc.JdbcProfile.API.Table[(MetadataModule.this)#Metadata]

    The 'METADATA' table contains all versioned metadata of all layers in the catalog.

    The 'METADATA' table contains all versioned metadata of all layers in the catalog. The versioning mechanism is implemented by means of two columns, 'FROM_VERSION' and 'TO_VERSION'. 'FROM_VERSION' contains the version of the catalog when the metadata entity was created (as a new entity is created when a partition changes between two versions, this is conceptually equivalent to the version when the metadata was "updated"). 'TO_VERSION' contains the last version of the catalog when the metadata was still valid. If 'TO_VERSION' is null the metadata is still valid.

    This allows the implementation of changes and deletes as: 1. When a partition changes, the existing metadata entity is updated (by writing down the version of the catalog before the commit into the 'TO_VERSION' column). A new metadata entity is then created with the updated information, with 'FROM_VERSION' equal to the new version of the catalog and 'TO_VERSION' equal to null.

    2. When a partition is deleted, its 'TO_VERSION' column is simply updated with the version of the catalog before the commit.
