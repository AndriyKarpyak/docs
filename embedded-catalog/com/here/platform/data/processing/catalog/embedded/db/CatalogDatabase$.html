  5. def apply(hrn: HRN): CatalogDatabase

    Retrieves the Database object of a catalog.

    Retrieves the Database object of a catalog. This function will get the catalog from the set of additional catalogs created via createDb (only catalogs created from the same JVM) and the set of catalogs created through the typesafe config configuration. In the latter case a new CatalogDatabase object (thus a new database connection) will be created for each Spark driver/ executor (if they are running on different JVMs). No serialization of the CatalogDatabase should be attempted. A given CatalogDatabase object should always be accessed from the same JVM where it was first created, but different CatalogDatabase can refer to the same database. If this library is used on a Spark cluster it's advisable to populate the databases with their schema in advance.

    hrn

    HRN of the catalog.

    returns

    The catalog database.

  8. def createDb(hrn: HRN, backend: String, url: String, layerConfigurations: Map[Name, LayerConfiguration], additionalBackendOptions: String*): Unit

    Programmatically creates a new catalog database.

    Programmatically creates a new catalog database.

    hrn

    HRN of the catalog.

    backend

    Database implementation (e.g. 'h2', 'sqlite', 'postgres').

    url

    JDBC URL of the database backend.

    layerConfigurations

    Layers' configuration.

    Note

    Unlike a constructor, this function does not return a local Database connection, but rather pushes a new Database connection into the current JVM's set of databases. Additional databases, unlike databases configured via application.conf, are local to the JVM they are created from. This function, like deleteDb, is meant for local development and testing.
