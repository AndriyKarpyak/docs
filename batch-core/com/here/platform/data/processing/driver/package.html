  9. sealed trait Executor extends InputLayers with InputPartitioner with OutputLayers with OutputPartitioner

    Base class for all the executors.

    Base class for all the executors.

    Note

    This policy established is the following: RDD passed to every execute function are guaranteed to be reusable multiple times efficiently without the need for the implementations to persist them. Implementations shall not persist passed RDDs. These are either already persisted by the framework or, anyway, guaranteed to be reusable multiple time efficiently. Implementations shall not therefore require() or assert() that passed RDDs are persisted, although it's guaranteed that they will be, or equivalent. RDDs returned by every execute function don't have to be persisted. They may be persisted if this is useful to the implementations, but they don't have to be. The framework may persist them once returned, if not already persisted.

    ,

    Executors work at RDD level, meaning that RDDs are passed and returned to the functions each executor implements. It is important to define a common policy regarding persistence of passed and returned RDDs. Not respecting this policy introduces a risk of Spark throwing an exception due to the fact that some RDDs may result to be persisted twice with different storage level (ACFC-372). This policy may be applicable to other classes than executors, such as com.here.platform.data.processing.compiler.NonIncrementalCompiler, com.here.platform.data.processing.compiler.DepCompilerBase and derived.
