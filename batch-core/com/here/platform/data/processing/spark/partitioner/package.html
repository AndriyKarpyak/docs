  1. case class LocalityAwarePartitioner(numPartitions: Int, level: Int) extends PartitionNamePartitioner with Product with Serializable

    Class implementing a Partitioner for com.here.platform.data.processing.catalog.Partition.Key that is aware of the geographic location of the keys and can therefore put keys that are geographically close to each other inside the same partition.

    Class implementing a Partitioner for com.here.platform.data.processing.catalog.Partition.Key that is aware of the geographic location of the keys and can therefore put keys that are geographically close to each other inside the same partition. This increases the data locality and speeds up processing of Spark worker nodes.

    This is implemented by detecting which com.here.platform.data.processing.catalog.Partition.Key are actually HERE tiles: in this case keys are grouped at a given quadtree level, that is generally higher (= lower number) that the level of the keys. Keys that are at a level even higher that the one specified, or keys that are not HERE tiles are partitioned using their hashcode and spread across all the available partitions. For them, data locality is not achieved.

    numPartitions

    the overall number of partitions

    level

    the grouping level in case of HERE tile partitions, as level of a quadtree

    Note

    this partitioner can be useful to both minimize data reshuffling in the context of Spark join and group operations and to introduce some data locality when compiling output tiles by having tiles close to each other in the same Spark partition
