  1. trait PartitionFilter extends AnyRef

    Filters partitions according to user-defined predicate.

    Filters partitions according to user-defined predicate.

    Partition filters allow to define just a subset of partitions to process and thus are useful in debugging. Currently only com.here.platform.data.processing.compiler.RefTreeCompiler is able to use partition filtering.

    To provide a custom predicate, the user has to do the following: 1. Implement the trait with a case class; implement predicate for filtering. 2. Add the constructor for that subclass that accepts a single JValue argument. Extract information from this argument however you like. The easiest way is to implement a case class containing your filter parameters and use com.here.platform.data.processing.driver.config.PartitionFilterConfig.extractParam helper method. 3. Add your subclass to your project's Kryo registrator so that Spark is able to serialize it.

    To enable filtering, add partitionFilter object to here.platform.data-processing.executors.debug with two fields: - className (string): a fully-qualified name of your own PartitionFilter subclass. - param (object): a relevant configuration for running your filter (e.g. partition list, or bounding box). The value of 'param' is passed as is to the constructor of your subclass.

    See also

    com.here.platform.data.processing.driver.config.PartitionFilterConfig, PartitionListFilter, PartitionListFilterParam
